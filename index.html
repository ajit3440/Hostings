1. Introduction

This module uses a sample .NET MAUI application called TipCalculator. The app is used in the hospitality industry to quickly calculate tips and totals. Your goal is to make the UI easy to update whenever branding changes (colors, fonts, styles).

To achieve this, you use:

  • Resources – shared values for colors, sizes, etc.
  • Dynamic resources – values that can update the UI at runtime.
  • Styles – reusable appearance definitions for controls.
  • Application-wide resources – definitions shared across all pages.

By the end of the module, you can centralize UI definitions and avoid repeated hard-coded values in your XAML.

2. Define and use resources

A resource is any reusable object you define once and reference from multiple places in your UI. Typical resources include colors, font sizes, styles, and platform-specific values.

Example: Color resource
<ContentPage.Resources>
    <ResourceDictionary>
        <Color x:Key="PageControlTextColor">Blue</Color>
    </ResourceDictionary>
</ContentPage.Resources>

Using the resource:

<Label TextColor="{StaticResource PageControlTextColor}" />
Intrinsic type resources
<ContentPage.Resources>
    <x:Double x:Key="fontSize">22</x:Double>
    <x:Boolean x:Key="isEnabled">True</x:Boolean>
    <x:String x:Key="welcome">Hello</x:String>
</ContentPage.Resources>
Platform-specific resource
<OnPlatform x:Key="textColor" x:TypeArguments="Color">
    <On Platform="iOS" Value="Silver" />
    <On Platform="Android" Value="Green" />
    <On Platform="WinUI" Value="Yellow" />
    <On Platform="MacCatalyst" Value="Pink" />
</OnPlatform>

3. Exercise: Use page-level resources

Goal: Remove repeated color and size values in StandardTipPage.xaml.

  1. Open StandardTipPage.xaml from the exercise1/TipCalculator project.
  2. Identify repeated values:
    • BackgroundColor="Silver" on the main Grid.
    • TextColor="Navy" and FontSize="22" on several labels.
  3. Add resources to the page:
<ContentPage.Resources>
    <ResourceDictionary>
        <Color x:Key="bgColor">#C0C0C0</Color>
        <Color x:Key="fgColor">#0000AD</Color>
        <x:Double x:Key="fontSize">22</x:Double>
    </ResourceDictionary>
</ContentPage.Resources>
  1. Use the resources with StaticResource:
<Grid x:Name="LayoutRoot"
      BackgroundColor="{StaticResource bgColor}"
      Padding="10">
<Label Text="Bill"
       TextColor="{StaticResource fgColor}"
       FontSize="{StaticResource fontSize}" />

Run the app – the UI should look the same, but the XAML is cleaner.

4. Use and update dynamic resources

DynamicResource works like StaticResource, but also listens for changes when the resource value is updated at runtime.

Defining and updating a resource
<ContentPage.Resources>
    <Color x:Key="PanelBackgroundColor">Blue</Color>
</ContentPage.Resources>
// code-behind
Resources["PanelBackgroundColor"] = Colors.Green;
Using DynamicResource
<StackLayout BackgroundColor="{DynamicResource PanelBackgroundColor}">
    ...
</StackLayout>

When the resource value changes, the UI is updated automatically.

5. Exercise: Use dynamic resources to update elements

Goal: Implement Light/Dark theme using dynamic resources.

  1. Open StandardTipPage.xaml.cs.
  2. Replace the code that sets properties on individual controls with code that updates resources:
void OnLight(object sender, EventArgs e)
{
    Resources["fgColor"] = colorNavy;
    Resources["bgColor"] = colorSilver;
}

void OnDark(object sender, EventArgs e)
{
    Resources["fgColor"] = colorSilver;
    Resources["bgColor"] = colorNavy;
}
  1. In StandardTipPage.xaml, change color bindings to use DynamicResource:
<Grid x:Name="LayoutRoot"
      BackgroundColor="{DynamicResource bgColor}"
      Padding="10">
<Label Text="Bill"
       TextColor="{DynamicResource fgColor}" />

Run the app and verify that the Light/Dark buttons change the theme at runtime.

6. Create a consistent UI by using styles

Styles let you group multiple property settings into a reusable definition, applied to a specific control type.

Style example
<Style x:Key="MyButtonStyle" TargetType="Button">
    <Setter Property="BackgroundColor" Value="#2A84D3" />
    <Setter Property="BorderColor" Value="#1C5F9B" />
    <Setter Property="BorderWidth" Value="3" />
    <Setter Property="TextColor" Value="White" />
</Style>
<Button Text="OK" Style="{StaticResource MyButtonStyle}" />
Implicit style
<Style TargetType="Button">
    <Setter Property="BackgroundColor" Value="Blue" />
    <Setter Property="TextColor" Value="White" />
</Style>

This style is applied automatically to all buttons on the page.

Apply to derived types
<Style TargetType="Button" ApplyToDerivedTypes="True">
    <Setter Property="BackgroundColor" Value="Black" />
</Style>

7. Exercise: Create and apply a style

Goal: Use a style to set 22pt Bold for multiple labels.

  1. Add a style to StandardTipPage.xaml in the page resources:
<Style x:Key="infoLabelStyle" TargetType="Label">
    <Setter Property="FontSize" Value="{StaticResource fontSize}" />
    <Setter Property="FontAttributes" Value="Bold" />
</Style>
  1. Apply this style to billLabel, tipLabel, totalLabel and remove their explicit FontSize and FontAttributes values.
  2. Run the app – it should look the same.
Base style and inheritance

Create a base style for labels:

<Style x:Key="baseLabelStyle" TargetType="Label">
    <Setter Property="FontSize" Value="{StaticResource fontSize}" />
</Style>

Update infoLabelStyle to inherit from the base:

<Style x:Key="infoLabelStyle"
       BasedOn="{StaticResource baseLabelStyle}"
       TargetType="Label">
    <Setter Property="FontAttributes" Value="Bold" />
</Style>

8. Create and use application-wide resources

To reuse resources and styles across multiple pages, define them in App.xaml in the application-level resource dictionary. 

<Application.Resources>
    <ResourceDictionary>
        <ResourceDictionary.MergedDictionaries>
            <ResourceDictionary Source="Resources/Colors.xaml" />
            <ResourceDictionary Source="Resources/Styles.xaml" />
        </ResourceDictionary.MergedDictionaries>

        <Color x:Key="bgColor">#C0C0C0</Color>
        <Color x:Key="fgColor">#0000AD</Color>
        <x:Double x:Key="fontSize">22</x:Double>

        <Style x:Key="baseLabelStyle" TargetType="Label">
            <Setter Property="FontSize" Value="{StaticResource fontSize}" />
        </Style>
        <Style x:Key="infoLabelStyle"
               BasedOn="{StaticResource baseLabelStyle}"
               TargetType="Label">
            <Setter Property="FontAttributes" Value="Bold" />
        </Style>
    </ResourceDictionary>
</Application.Resources>

Now all pages in the app can reference bgColor, fgColor, fontSize, baseLabelStyle, and infoLabelStyle.

9. Exercise: Use application-wide resources

Goal: Share label styles between StandardTipPage and CustomTipPage.

  1. In CustomTipPage.xaml, set the style for billLabel:
<Label x:Name="billLabel"

10. Summary

In this module, you improved the TipCalculator UI and made it easier to support future branding changes by using:

  • Resources to centralize colors, sizes, and values.
  • Dynamic resources to update the UI at runtime (for example, theme switching).
  • Styles to define control appearance consistently.
  • Application-wide resources to share definitions across multiple pages.

These techniques let you build maintainable, consistent, and easily themed .NET MAUI applications.
